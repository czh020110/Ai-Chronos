# 目标边界

## 总体目标

构建一个结构化 AI 时空知识图谱平台，以高交互美学时间线为主轴，通过多智能体实现海量文献自动萃取，利用混合 RAG 提供史料级精准检索，并依靠事实核查智能体与社区协同形成可持续演进的平台自治能力，解决 AI 发展信息碎片化、传统维基更新滞后、搜索噪音大和高质量内容平台人工审核成本高的问题。

## 当前阶段目标

- [ ] 建立离线事件采集与入库能力：验收方式：使用 Python 脚本完成维基百科与 AI 里程碑事件抓取、清洗并灌入数据库，可通过人工抽查确认入库结果正确。
- [x] 完成高交互时间线 MVP 展示：验收方式：前端实际运行后可在桌面端钻取浏览年/月/日三层时间线，拖拽和滚轮滚动带 RAF 惯性效果，Canvas 星空与宇宙河流背景渲染流畅。
- [x] 完成 Markdown 事件详情抽屉：验收方式：点击时间线节点或日事件球后可弹出右侧详情抽屉，正确渲染 Markdown 事件内容（含标题、列表、标签、影响力进度条与来源链接）。
- [ ] 明确第一阶段内容边界与采集范围：验收方式：首批展示内容覆盖 AI 顶会核心论文、主流开源模型发布和标志性商业新闻，来源限定在 Arxiv、HuggingFace 与主流 AI 媒体。
- [ ] 落实状态机驱动的数据发布约束：验收方式：阶段方案和后续实现均遵循 Draft→Staged→Published 的流转，不直接覆盖线上数据。

## 成功标准

- [x] 第一阶段可交付一个基于已入库事件的静态时间线演示版本，支持时间浏览与详情查看。（前端 UI 部分已完成，使用示例数据驱动；后端离线入库待实现）
- [x] 第一阶段交付严格限定在离线采集、静态展示和详情阅读，不提前实现第二阶段动态抓取与 RAG、第三阶段社区自治能力。
- [x] 所有事件内容存储与前端渲染统一使用纯 Markdown 格式。
- [x] 时间线在大量事件节点场景下仍保持可接受的缩放与动画流畅度。
- [x] 第一阶段方案满足 AI 垂直领域边界，不退化为全网泛科技新闻聚合产品。

## 非目标

- 不做全网泛科技新闻聚合，内容范围严格限定在 AI、大模型与前沿计算领域。
- 不提供整篇论文深度翻译与源码解析，仅保留摘要层面的精炼提炼。
- 不完全抛弃人工干预，争议事件和低置信度结果仍保留给管理员仲裁。
- 第二阶段的动态 Spider、Distillation Agent、RAG 混合检索能力暂不纳入第一阶段交付。
- 第三阶段的社区提交、事实核查闭环、后台运营看板暂不纳入第一阶段交付。

## 质量底线

- 测试底线：阶段验收至少覆盖离线入库抽查、时间线缩放交互、节点详情渲染，并保持 Abstract-only 策略与有限抓取频率的成本约束不被突破。
- 安全底线：所有后续发布链路都必须遵循来源可追溯、低置信度结果进入人工仲裁、Staged 状态不直接覆盖线上数据的约束。
- 性能底线：前端时间线需支持大量事件节点下的流畅缩放、聚合与动画展示。
- 文档底线：所有事件内容存储和前端渲染必须是纯 Markdown 格式，阶段边界与验收口径需持续与实现保持一致。

## 推进节奏

- 每个阶段都必须形成可验证的阶段成果，先完成可演示 MVP，再继续扩展下一阶段能力。
- 第一阶段完成静态时间线展示与离线入库闭环后，再进入第二阶段的动态 Agent 与 RAG 建设。
- 第二阶段的检索与自动化入库链路稳定后，再开放第三阶段的社区提交与自治流程。
- 每次推进都遵循状态机驱动的数据流转，默认按 Draft→Staged→Published 逐步放行，不直接操作线上数据。

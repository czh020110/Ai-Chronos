# 项目目标

## 总体流程目标

构建一个以高互动时间线为核心载体的 AI 进化协同百科平台：先通过第一阶段 MVP 完成离线采集、清洗、入库与静态展示闭环，再在第二阶段引入自动抓取与混合 RAG 检索，最后在第三阶段接入社区贡献、事实核查与后台仲裁，逐步形成可持续演进的结构化 AI 时空知识图谱。

## 核心功能目标

- [x] 全景无级时间线：验收方式：前端实际运行后可钻取浏览年/月/日三层视图，Canvas 宇宙背景动画流畅，节点 Spring 入场动画平滑，拖拽/滚轮滚动带 RAF 惯性效果。
- [x] Markdown 事件详情抽屉：验收方式：点击时间线节点后弹出右侧抽屉，正确渲染 Markdown 内容（含标题、列表、加粗、链接、引用块），显示标签、影响力进度条、来源链接。
- [ ] RAG 智能检索入口：验收方式：输入自然语言问题后可返回相关事件并生成总结，时间线高亮相关节点。
- [ ] 社区交互台：验收方式：用户可提交新事件和报错反馈，状态按设计正确流转。
- [ ] Agent Ops 监控盘：验收方式：后台面板可查看 Agent 运行日志、API 消耗与成功率等指标。
- [ ] 数据仲裁台 Human-in-the-loop：验收方式：低置信度贡献会正确进入待审批列表，管理员可放行或驳回。
- [ ] 多智能体系统：验收方式：采编链路与核查链路按设计状态机正确执行，支持 Draft、核查、合并或驳回等流转。

## 阶段功能范围

- 第一阶段（MVP）：目标：完成维基百科与 AI 里程碑事件的离线抓取、清洗入库，交付桌面端优先的高美感时间线展示与 Markdown 详情抽屉；状态：进行中（前端 UI 已交付，后端离线入库待实现）；依赖：离线 Python 脚本、数据库基础结构、Next.js 前端时间线组件。
- 第二阶段（动态注入与 RAG）：目标：上线 Spider 与 Distillation Agent 的每日自动抓取提炼入库流转，并提供基于 pgvector、BM25 与重排序的混合检索入口；状态：未开始；依赖：第一阶段稳定的数据结构与展示闭环、PostgreSQL 检索能力、Agent 调度链路。
- 第三阶段（社区自治）：目标：开放前台提交与报错入口，接入 Triage、Fact-Checker、Merge 工作流，并补齐后台审批与运营看板；状态：未开始；依赖：第二阶段稳定的状态机与检索基础、管理员审批流程、限流与安全策略。

## 实现发现

- 项目采用三阶段递进路线，先验证静态惊艳的时间线体验，再逐步扩展自动化抓取、检索与社区自治能力。
- 第一阶段的内容边界已明确限定为 AI 顶会核心论文、主流开源模型发布和标志性商业新闻，不覆盖全网泛科技资讯。
- 平台长期质量依赖状态机驱动的数据流转、来源引用与人机协同仲裁，不能让自动化链路直接覆盖线上数据。
- 前端时间线已实现钻取式导航（year → month → day），采用 Canvas 2D 宇宙背景 + DOM 节点 + Framer Motion Spring 动画，替代最初的无级缩放方案。钻取式交互在大量事件场景下更直观、性能更可控。
- 前端使用静态示例数据 `frontend/lib/data.ts` 驱动展示，包含 2012-2026 年 AI 里程碑事件（AlexNet、GAN、GPT 系列、DeepSeek 等）；后续需替换为后端 API 数据源。
- 前端支持三种视图模式（时间线/网格/列表），通过 ViewToggle 组件切换；搜索面板支持关键词匹配和键盘导航（上下箭头 + Enter）；Ctrl/Cmd+K 快捷键打开/关闭搜索。
- 前端 UI 已完成一轮视觉重构：动态背景从普通星空升级为"认知地层/神经星图"复合背景（星云粒子、流动极光带、扫描光效）；新增左侧项目叙事面板（统计摘要）和右侧当前坐标信息面板；时间线视觉升级为"纪年星图 / 月度透镜 / 日级轨迹"三层风格；全局玻璃质感与信息层级进一步强化。

## 后续目标调整

- 需要在后端实现阶段确认数据 API 入参/出参格式，与前端 AIEvent 类型保持一致。
- 需要在第二阶段开始前细化 PostgreSQL、pgvector、倒排索引与 BGE-Reranker-v2 的检索组合与验收方式。
- 需要在第三阶段开始前补齐社区身份、限流、防 Spam 和后台审批口径的详细规则。
- 移动端触摸拖拽和响应式适配需在后续迭代中跟进（当前仅适配桌面端鼠标交互）。

# 核心数据流

## 前端钻取式时间线交互链路

### 本次更新原因

- 关联 Commit：初始前端交付
- 更新原因：首版完整前端时间线 UI 实现，需要记录核心交互数据流。

### 链路目标

用户在年/月/日三层视图间钻取浏览 AI 事件时间线，点击节点查看 Markdown 详情，通过搜索面板快速定位事件。

### 修改前数据流

新增链路，修改前无。

### 修改后数据流

[页面入口 `app/page.tsx`] → [状态管理 (drill, selectedEvent, searchOpen)] → [组件渲染]
  → [NavBar: 触发搜索打开]
  → [Timeline: 根据 drill 渲染年/月/日节点] → [节点被点击] → 分支：
    ├─ 年份/月份节点 → [onDrillDown] → [更新 drill 状态] → [Timeline 重新渲染下一层级]
    └─ 日节点/事件球 → [onNodeClick] → [更新 selectedEvent + drawerOpen] → [DetailDrawer 显示 Markdown 详情]
  → [SearchPanel: 打开搜索面板 → 输入关键词 → 选择命中事件] → [onSearchSelect] → [打开 DetailDrawer + 高亮对应节点]

### 输入

- `AIEvent[]` 事件数组
  - 来源：`frontend/lib/data.ts` 中的 `sampleEvents`（静态示例数据）
  - 关键字段：id, title, event_date, content_md, tags, source_urls, impact_score, category, status
  - 约束：第一阶段无后端，事件数据在组件初始化时一次性加载

### 处理步骤

1. `frontend/app/page.tsx`::`Home`
   - 职责：顶层状态容器，管理 drill、selectedEvent、drawerOpen、searchOpen、viewMode、highlightedId
   - 调用方：Next.js App Router（`layout.tsx` → `page.tsx`）
   - 被调用方：StarField, NavBar, Timeline, DetailDrawer, SearchPanel, DrillControls, ViewToggle
   - 调用时机：页面首次渲染
   - 输入：无外部输入，使用 `useState<AIEvent[]>(sampleEvents)` 初始化
   - 输出：通过 props 向子组件分发状态与回调
   - 状态变化：drill 状态通过 `handleDrillDown` / `handleDrillUp` 切换层级；searchOpen 通过 `Ctrl/Cmd+K` 快捷键或 NavBar 按钮切换
   - 异常分支：无

2. `frontend/components/timeline/Timeline.tsx`::`Timeline`
   - 职责：根据 `drill.level` 计算布局节点（年份间距 160px，月份按 `winW/14`，日期按 `winW/(daysInMonth+4)`），渲染 Canvas 背景和 DOM 节点
   - 调用方：`app/page.tsx::Home`
   - 被调用方：TimelineCanvas（Canvas 2D 背景）、GridView、ListView
   - 调用时机：页面渲染、drill 层级切换、viewMode 切换
   - 输入：events（完整事件列表）、drill（当前钻取状态）、viewMode（timeline/grid/list）、onNodeClick、onDrillDown、highlightedId
   - 输出：Framer Motion 动画节点（Spring 入场动画）、Canvas 背景（宇宙河流辉光、刻度线、粒子、当前时刻标记）
   - 状态变化：内部维护 `scrollX`（RAF 惯性滚动）、`hoveredNode`（Tooltip 显示）、`transitioning`（层级切换标记）
   - 异常分支：viewMode 非 "timeline" 时直接渲染 GridView 或 ListView

3. `frontend/components/ui/DrillControls.tsx`::`DrillControls`
   - 职责：底部面包屑导航，显示当前层级路径（全部年份 / 2024年 / 1月），提供"返回上级"按钮
   - 调用方：`app/page.tsx::Home`
   - 被调用方：无
   - 调用时机：drill 状态不为 "year" 时渲染返回按钮
   - 输入：drill, onDrillUp
   - 输出：层级指示点（三点分别对应年/月/日，当前层金色发光）
   - 异常分支：无

4. `frontend/components/search/SearchPanel.tsx`::`SearchPanel`
   - 职责：搜索面板，支持关键词匹配（标题/标签/内容），键盘导航（上下箭头 + Enter）
   - 调用方：`app/page.tsx::Home`
   - 被调用方：无
   - 调用时机：isOpen 为 true 时渲染
   - 输入：events, isOpen, onClose, onSelect
   - 输出：最多 8 条搜索结果（含标题、日期、标签、影响力分数）；选中后调用 onSelect
   - 状态变化：内部维护 query 和 selectedIdx
   - 异常分支：无匹配时显示"未找到相关事件"

5. `frontend/components/drawer/DetailDrawer.tsx`::`DetailDrawer`
   - 职责：右侧滑入式 Markdown 详情抽屉，渲染事件摘要、标签、影响力条、参考来源
   - 调用方：`app/page.tsx::Home`
   - 被调用方：react-markdown + remark-gfm
   - 调用时机：drawerOpen 为 true 且有 selectedEvent 时渲染
   - 输入：event, isOpen, onClose（Esc 键关闭）
   - 输出：Markdown 渲染内容、影响力进度条动画、来源链接列表
   - 异常分支：event 为 null 时不渲染

6. `frontend/components/background/StarField.tsx`::`StarField`
   - 职责：200 颗星空粒子背景动画（Canvas 2D，RAF 循环）
   - 调用方：`app/page.tsx::Home`
   - 被调用方：无
   - 调用时机：页面渲染即启动，resize 时销毁重建（200ms 防抖）
   - 异常分支：无

### 状态流转

- 钻取路径：`year` 级（全部年份）→ 点击年份 → `month` 级（该年 12 个月）→ 点击月份 → `day` 级（该月每日视图）→ 点击日节点 → 打开 DetailDrawer
- 返回路径：点击 DrillControls "返回上级" 逐级退回 year 级
- 搜索路径：打开搜索面板 → 输入关键词 → 选择结果 → 打开 DetailDrawer 并 3 秒高亮对应节点 → 高亮自动消退

### 输出

- 用户可见：年/月/日钻取时间线（Canvas 背景 + DOM 节点 + Spring 入场动画）、Markdown 详情抽屉、网格/列表视图
- 用户不可见：无后端写入、无数据持久化（当前阶段）

### 错误分支

- 搜索无结果：显示"未找到相关事件"提示，不影响页面其他交互
- 空月份/空日期节点：渲染为半透明虚节点，不可点击，标签显示无事件

- `frontend/lib/types.ts`：AIEvent, DrillState, ViewMode, ZoomLevel 等同构类型定义
- `frontend/lib/data.ts`：sampleEvents 静态示例数据（首批 AI 里程碑事件）
- `frontend/lib/tagStyles.ts`：tagClass 样式映射表
- `frontend/components/timeline/Timeline.tsx`：时间线/网格/列表三合一视图组件
- `frontend/components/ui/DrillControls.tsx`：底部面包屑钻取导航
- `frontend/components/ui/ViewToggle.tsx`：视图切换（时间线/网格/列表）
- `frontend/components/ui/ZoomControls.tsx`：缩放控制按钮（预留，当前由钻取导航替代）
- `frontend/components/search/SearchPanel.tsx`：搜索面板
- `frontend/components/drawer/DetailDrawer.tsx`：Markdown 详情抽屉
- `frontend/components/navigation/NavBar.tsx`：顶部导航栏
- `frontend/components/background/StarField.tsx`：星空背景动画
- `frontend/tailwind.config.ts`：TailwindCSS 主题配置（cosmos 色系、字体族、自定义动画）
- `frontend/next.config.js`：Next.js 配置
- `frontend/tsconfig.json`：TypeScript 配置

### 验证方式

- `cd frontend && npm run build`：构建通过，无类型错误
- `cd frontend && npm run dev`：开发服务正常启动，时间线可交互钻取、搜索、查看详情
